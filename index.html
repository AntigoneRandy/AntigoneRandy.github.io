零杯参赛感想

和自强的两位同学组队参加了零杯取得了一等奖的好成绩，写这篇博客来简单记录一下我的参赛感想，也对后来人提出一些建议。

11月11日就收到了和零杯参赛相关的推送，但是当时还有自强的任务在身，于是先没有太在意。也有几位自强的大佬来找过我组队，不过都妹有后续了qwq

后来和小翁在c语言课上达成了共识，这说明线下面基很重要 同时小翁正好邀请了一位之前找过我组队的巨巨lsx，于是我们三人小队就这样组好力（）

我们组开了一次会过后决定了做游戏相关的网页。最开始想的是做游戏产业与电子竞技，林巨巨很快画好了图，看着蓝湖上的图我感叹终于有一个审美在线的UI了。之后就是找素材、复刻，我个人负责的是主页面的代码编写，除了那个显示屏的参数调的我欲仙欲死之外，总体没遇上什么障碍。当然难度还是不小，因为我本来也把这次比赛当成一个锻炼自己（骗钱）的机会，就没有用现成的库和插件，所有代码都是手写的，最后写了2k行，感觉还行。最开始我们代码都是我写的，后来发现有些小功能确实不好整（比如从兰博基尼官网抄来的轮播图设计和瀑布流、时间轴两个版块），于是便把锅分了出去。同时代码的bug也是大家一起改，所以林巨巨不满足于压缩包在群里传来传去的低效，决定改用github。

可惜我和小翁都不太会，想我之前暑假还学过git和github的用法，可惜这几个月以来除了Download Zip以外也没用到其他功能，知识和指令早就忘得一干二净了。这充分说明东西不是学会是用会的。。。由于时间紧迫我也只速成了git pull 和git push。中间还因为好几次误操作把本地的commit搞炸了，感谢林巨巨包容（）好在我们代码风格比较能看，我也确实有写注释的习惯，所以还没出现“你这行代码写的是个啥”这种问题hhh

颁奖搞完过后就溜去自强的项目展示了。那个项目我属实写的不够认真，给组内成员们道歉qwq所幸最后展示效果还行，没被骂嘻嘻。

今天还得到了前端组最佳新人奖，可惜iPad又又又又又又又摔坏了。唉，一半欢喜一半忧吧。。

最后给以后参加零杯的学弟学妹们说几点答辩的注意事项和整体设计思路上的问题吧：

  1. 主题一定要明确。你做网站是要干什么，你面对的用户群体是谁，这些都是队员们需要思考的问题。我们最开始的主题是“游戏产业与电子竞技”，但是后面发现这两者不好融合，于是决定把游戏做为引子，来引出电子竞技的主题。因此我们采用了递进的关系来构造我们的作品。后面的答辩结果也证明我们的决定是正确的。

  2. 网站要有逻辑。做网站不是想到什么写什么，最好有一个清晰的骨架把所有内容串联起来，并且每一个部分都要有它的主题和意义所在，有头还要有尾。这个在做网站的时候就要有所思考。我们在答辩的时候采用了前文所述的递进层级，对我们的网站做了一个比较清晰的解说。因为评委老师作为观众，是根据你说的内容来评判的。如果你说的话很有逻辑、能把整个网页的设计理念和实际功用说明清楚，那你就成功了一半了。同样，对于页面的每个功能也是如此，要有详细的解说，不是一句带过的。

  3. 不要把网站做的过于花里胡哨，空有炫酷。这也是前端初学者很容易犯的问题。刚进入前端世界可能热衷于各种炫酷的特效和动画，下载个插件写几行代码，或者直接调个api，短时间看可能觉得很nb，但是实际上这样的意义不大。个人不建议使用太多的插件和库（我们这次的项目基本上都是纯手写，只用了一些情理之中的库）。说实话，评委老师不是傻子，你调用插件的页面炫酷无比，自己写的部分丑陋不堪，前后页面风格明显不搭，哪些是自己写的哪些是调个插件填个参数就完事的一眼便知。在内行看来更是如此。插件应当是为了简化开发、提高效率而用，如果你还没有手造这个轮子的实力，那建议不要用它。

  4. 切忌面向展示编程。在场的老师和同学都是有提问权的。如果你写几个按不了的按钮/只写个登录界面就说自己有登录功能/用假的返回值骗过展示甚至直接套网上的模板，会死得很惨。而且这是诚信问题，建议不要这样搞。

  5. 搞清楚零杯的意义是什么。零杯不是一个炫技的地方，这个比赛设立的初衷是让同学们以练代学，在练习中学习。因此你要体现出你为了比赛学习的东西，如果能在项目中代入你在课上学习的东西那就更好了。我们组在这次答辩中靠着这个思路获得了很大的优势。比如猜人小游戏中的贪心，以及一些简单的性能优化，其实这些都是很简单容易的东西，但是能让评委老师看到你为这个项目做出的努力，在大一的时候就能有一个这样的学习思维，也是很好的。（决策树我本来是会的，故意丢给了没学过的组员去做，展示的时候更加纯真哈哈哈哈）要抱着一种虚心求学的态度来答辩。除此之外，我们还提到了代码可重用性、组件化，总之要记住的一点就是作为网页开发设计的初学者，参加零杯是为了学习，为了给未来的项目铺路。围绕这一点来展开你们的项目，评委老师的态度会好很多。

  6. 分工要明确。这个和选队友也是息息相关的，建议把整个网页分成不同的组件，每个人完成一部分的开发再合并。答辩的时候完成该部分的人上去，这样显你们团队合作井井有条。

  7. 要有亮点。要有一个或者两个比较有意思、能激起评委老师和其他观众兴趣的点。比如我们这次的亮点放在了猜人游戏和显示屏放大缩小上面。这里有一个小技巧是在展示的时候把亮点轻描淡写地带过，老师必然追问，这样就可以用答辩而不是展示的时间来详细阐释你们的内容（所以亮点一定要够亮，让老师有追问的兴趣）。比如猜人小游戏的算法就让老师很感兴趣，第一个问题就问的这个。另外有个提醒是如果涉及算法的内容实现方法的难度不能太高或太低，还是拿小游戏举例子，如果难度太低比如写一堆if/else（事实上我们组员真提过这个方案），会让老师刚提起的兴趣马上消散，对你们的底子也一清二楚了。如果难度太高，比如真的写个机器学习决策树，逼格是上去了，但是一来评委老师参与度会比较低（如果老师不是这个方向的很可能不了解你讲的算法），二来体现不出学习的态度（这种多半都是之前就会的，甚至评委老师有可能会觉得你们是抄的）。效果不见得会很好（这真的是大一新生能做出来的水平吗？）。

狗狗银行-题解

发表于 2020-11-08

狗狗银行

题目来源：USTC-Hackergame 2020

你能在狗狗银行成功薅到羊毛吗？

基本思路

进入页面后稍做观察，发现不能通过常规的借钱-还钱-利率差的方式盈利，因为每日吃饭要花去10元，而信用卡利率大于储蓄卡利率，不可能通过常规方式获胜。

注意到利息计算没有小数，而是四舍五入到整数位，考虑从此处入手薅羊毛。即储蓄卡中有167元时，167*0.3%=0.501>0.5，四舍五入可以获得1元的利息，1000元可以办卡约5张，获利5元，大于原本的3元即实际利率变成了1/167=0.59%，高于信用卡的利息5%。因此当储蓄卡足够多时候，可以有可观的盈利。此方法是可行的。

稍加计算可以发现：

信用卡：一张a元，支出a*0.5%

储蓄卡：x张，每张167元，盈利x元

每日吃饭：10元

列一个二元一次方程组可解：

x * 167 = a + 1000

x - 10 - a * 0.5% > 1000

在满足此要求的情况下，恰饭一天即可盈利1000元+

容易解得一个x为6100，此时a为1017700。此时的办卡数超过了题目1000张办卡的限制（当然笔者在增加此限制之前已经用此思想把 flag 拿到手了）。要达到至多1000张办卡的条件限制，方法也很简单：由于每天净赚 0.09% 利息，即使办卡少，减去利息后也依旧有客观的收入（假设办805张卡，借钱800 * 167 = 133600元，资产共134600元，每日收入805元，指出10 + 133600 * 0.5% = 678元，也依旧有127元/天的收入），每天逐渐就能积累到 1000 狗狗币。

但是这种方法有一个缺陷就是，每日恰饭后会产生不能盈利的利息，即167元和168元都是1元利息，而信用卡中的利息却能有效增加，一天后（133600 + 668）* 0.5% =671.34元，如果胜利条件要求的金额比较高，会导致盈利慢慢变少，每张卡200元时恰好不能盈利，之后就开始亏钱。因此正确的做法是每隔一段时间就将所有储蓄卡中的多出的钱全部收集到一起，或者直接还给信用卡。当然，经过测试，由于办卡数还是足够多，还未达到亏损的界限就已经胜利，用不着这么麻烦，只需要多恰饭几次即可。

点击“办新卡”，打开SwitchyOmega的burpsuite代理，点击办新的储蓄卡，打开burp中的Intercept发现抓包结果

发现向host/api/create接口发送了{"type":"debit"}的JSON数据包。

再用相同的方法抓包转账时的数据包，例如储蓄卡1向储蓄卡2转账500元时， burpsuite 抓包结果如下：

联想英语知识容易看出， src 是 source ，即付款卡， dst 是 destination ，即收款卡， amount 就是数额。

后面的事情就很简单了，先人工操作办一张信用卡，借133600元到1号储蓄卡，再办卡800张，平均每张卡分配167元。办卡和转账的过程可以使用 burpsuite 自带的爆破功能（Intruder）完成。

burpsuite爆破方法讲解

先找到刚才发办卡请求那个包，右键，点击Send to Intruder，之后点击Intruder。

Attack type讲解：

  • Sniper ：Sniper就是狙击手，他的攻击方式是：只有一个字典，先用该字典遍历第一个变量位置，再用该字典便利第二个变量位置，以此类推。没有被遍历的位置则保持原始的数据
  • Battering ram ：“重锤”，它的攻击方式是：同样只有一个字典，不过每次攻击都用这个字典的一项来代替所有的变量位置，第二次再用第二项代替所有的位置，以此类推
  • Pitchfork ：“草叉”，它一个变量对应一个字典，第一次攻击时，变量1对应字典1的第一项，变量2对应字典2的第一项，第二次攻击时，变量1对应字典1的第二项，变量2对应字典2的第二项，以此类推
  • Cluster bomb：“集束炸弹”，它也是一个变量对应一个字典，不过不同的是，变量1对应字典1的第一项后，变量2会依次取字典2的第1，2，3…项，直到字典2遍历结束后，变量1又会再取字典1的第二项，变量2再依次取字典2的第1，2，3…项，依此类推。

对于第一个，办卡，发现不用改变量，我们只需要重复800次即可。于是把debit设成变量，再用python生成800个debit，把它当为字典即可。

对于第二个操作，转账，只需要把dst改一下即可。我们随便选一个爆破方式，把dst的“2”设成变量，然后用python输出3-804的数字，把它作为payload（当然要把金额改成167元），attack结束后打开页面就会发现每张卡都有167元了，于是手动点恰饭10多次，即可达到目标获得flag。

如果要每隔一段时间就将所有储蓄卡中的多出的钱全部还给信用卡，也很简单。恰几次饭过后把金额设成多出来的钱，src设成变量，dst设成信用卡爆破一次即可。

自复读的复读机

能够复读其他程序输出的程序只是普通的复读机。
顶尖的复读机还应该能复读出自己的源代码。
什么是国际复读机啊（战术后仰）
你现在需要编写两个只有一行 Python 代码的顶尖复读机：

  • 其中一个要输出代码本身的逆序（即所有字符从后向前依次输出）
  • 另一个是输出代码本身的 sha256 哈希值，十六进制小写
    满足两个条件分别对应了两个 flag。
    快来开始你的复读吧~

输出自己代码的程序是很多程序设计书上面当作趣味题介绍的内容。实际上，这类程序有一个专门的名字：Quine

知识点补充： Python 中字符串格式化输出

Python 支持格式化字符串输出，写法和c语言类似（又称printf 风格的字符串格式化）：

其中常用的转义符有：%s输出字符串，%r输出带引号的字符串(也就是解释器能看懂，或者说能用eval()函数处理的字符串)，%d输出整数，%%输出单个%。

（实际上，%s的含义是使用 str() 转换任何 Python 对象，%r则为使用 repr() 转换任何 Python 对象）

“我真的只会开关机”

拿OI（信息学竞赛）举个例子，从初中甚至小学就开始学习信息竞赛的同学不在少数，也有人六年OI一场空的；也不乏有从高中才开始接触OI，然后一路高歌猛进拿下IOI（最高级的世界级OI比赛）金牌的。

一.好习惯

1. 不使用360，腾讯安全管家之类的国产安全软件

这类安全软件安全防护的本职工作效果很一般不说，而且会强制安装/通过各种隐蔽的方式诱导你安装它的全家桶和其他垃圾软件。而且像显示开机时间，自启“热点新闻”等等乱七八糟花里胡哨的功能会占用大量的系统资源，要不了多久电脑就会卡顿不堪

更有甚者，某安全软件在某些情况下会对编译后生成的exe文件误报毒并将其删除，使得编译运行过程无法正常运行，十分不利于学习计算机

况且咱们是信息安全的学生，只要做到少上不良网站，少下载未知来源的软件（包括部分含有木马的“破解版”软件），被感染的机率是非常小的。现在都0202年了，普通的病毒会被Windows自带的Windows defender拦截住，真正牛逼的病毒360这些也是防不住的，所以赶快把安全软件和它的全家桶卸载掉吧。

如果实在觉得没有个安全软件不放心，也可以试试火绒，国产安全软件中最良心的没有之一，平时很安静也不会捆绑安装，有些诸如联网控制，弹窗拦截等等功能还挺好用的

如果你是那种被各种问题缠的喘不过气，不想当工具人但又不好意思直接拒绝对方的“终身免费顾问”， 可以了解一下“期望值管理”的心理学技巧，这或许可以解决你的烦恼。

4. 下载软件时候请去官网，拒绝“xx软件园”，“xx资源网”

事实上，绝大多数软件的官网都是下载其最可靠的方式，没有之一（靠谱的博客或者个人网站也可以）。但是由于某黑心搜索引擎搞竞价排名恰烂钱，有很多诸如xxx软件园，xxx资源网的流氓网站就排在了搜索结果的前几位，而且部分软件的官网只有洋文界面，这会使人望而却步。流氓资源网站的套路在于进入过后会看到一个大大的“高速下载”，旁边在下面批注一句很小的“使用高速下载器”。点击这个按钮下载下来的东西不是你要的软件，而是它所谓的“高速下载器”。

“高速下载器”不仅不能把下载速度加快，还会在安装时用尽套路安装恶心人的捆绑软件，他们通过推广其他流氓软件的方式来盈利，流氓又会再安装流氓，一传十十传百，不久就会让你的电脑被流氓软件占领，大大拖慢系统运行效率和使用体验。

如果非要在这上面下载，请费心找找“普通下载”，可能下载下来的东西会阳间一点。

一.Markdown简介

什么是Markdown？

Markdown是一种轻量级标记语言，创始人为约翰·格鲁伯（英语：John Gruber）。 它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML（或者HTML）文档。这种语言吸收了很多在电子邮件中已有的纯文本标记的特性。

由于Markdown的轻量化、易读易写特性，并且对于图片，图表、数学式都有支持，目前许多网站都广泛使用Markdown来撰写帮助文档或是用于论坛上发表消息。 如GitHub、Reddit、Diaspora、Stack Exchange、OpenStreetMap 、SourceForge、简书等，甚至还能被使用来撰写电子书。——百度百科

看不懂对不对（初学的时候我也看不懂

说人话，Markdown是一种计算机语言，它非常适合用于写作（尤其是写博客）和记笔记（实在不想看到用笔抄代码的人了） 。

Markdown和直接用笔在笔记本上记笔记的重要区别在于它形式和内容分离，和传统手记“所写即所得”的方式不同，Markdown通过一些标记来实现特殊的效果，比如在源码里两个**中间括起来的内容会被加粗，用~~可以实现删除线等等。

为什么要使用Markdown？

由于它对于代码高亮的优秀支持，使得用它来记计算机相关课程的笔记变得简单，而最后的效果也十分美观。因此强烈推荐使用Markdown记计算机相关课程的笔记。

二.Markdown入门

编辑器

Markdown不像记事本，Word一样可以直接编辑，而是需要专门的编辑器来编辑。

本人使用的Markdown编辑器是vscode（需要安装插件以提高体验），这个网址链接到它的官网，你可以自己在网上寻找相应的下载配置教程，推荐这篇。 需要更多的内容，建议百度 Markdown编辑器以获得支持。

语法

前面说过Markdown的一大特点是形式和内容分离，因此它的使用方法需要单独学习。
